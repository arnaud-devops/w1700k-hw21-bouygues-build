## Checks passed

The CI checked the compile output, assembled rootfs, target kernel
configuration and generated DTB before publishing. A separate local audit then
downloaded the release assets and validated them again:

- OpenWrt checksums, `profiles.json`, target, profile, board and image size;
- GitHub build-provenance attestations for all seven release assets;
- FIT kernel, DTB, rootfs and sysupgrade metadata;
- `ramoops@86ff0000`, with the expected 64 KiB reservation;
- package manifest and CycloneDX SBOM policy;
- exact builder, source, feed and patch provenance;
- patch `971` present and fallback patch `965` absent;
- strict RTL8261CE driver, MT7996 and EN7581 NPU firmware payloads;
- full `wpad-openssl`, `dnsmasq-full`, `dnsproxy` and OpenSSL LuCI stack;
- normal dual-stack fw4 offload rule;
- DoQ and Wi-Fi sysupgrade recovery package;
- FlowSense, NPU, MLO/Wi-Fi 7, Watchcat, irqbalance, SQM/Cake, pstore,
  SFTP and the selected diagnostic tools;
- BusyBox `flock`, Watchcat serialization and conservative service defaults;
- absence of patch `965`, `bridger`, `wpad-basic-*`, generic Attended
  Sysupgrade, `owut`, `ttyd` and the LuCI file manager.

The seven published files are the sysupgrade ITB, manifest, CycloneDX SBOM,
`sha256sums`, `profiles.json`, `config.diff` and `build-info.txt`. No backup,
raw capture or tar archive was published.

On the default case-insensitive macOS filesystem, standard iptables files such
as `xt_DSCP.ko` and `xt_dscp.ko` collide during rootfs extraction. The local
auditor uses `unsquashfs -force` for that host-only extraction step. The
case-distinct files remain intact in the SquashFS image and the Linux CI
extraction passed normally.

## Remaining hardware gate

This audit proves reproducibility and image contents; it does not prove runtime
behavior. The image has not been flashed. The active router remains on the
validated Gilly 17.07 image.

An explicitly authorized hardware test is still required for boot and
sysupgrade recovery, both RTL8261CE links, WAN 10G, LAN2 multigig, sustained
IPv4/IPv6 PPE traffic, all three APs, DoQ, Watchcat, pstore and the irqbalance
comparison. Until then, the v2 prerelease must not be treated as stable.
