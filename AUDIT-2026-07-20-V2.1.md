## Corrected findings

The three findings from the first v2 audit are resolved:

- the RTL8261CE callback now requires Clause 45 and
  `genphy_match_phy_device()` before Gilly's PMA/PMD model `0x9` check;
- Wi-Fi recovery retains `flock`, queries every real `hostapd.*` ubus object,
  requires `status=ENABLED`, maps its AP name to `/sys/class/net`, and waits
  for the AP netdev to be operational;
- FlowSense `npu-jitter` is installed but disabled by default, and the init
  script starts the two-second external probe only after explicit UCI opt-in.

The package revisions carrying the latter two changes are
`w1700k-hw21-bouygues-support 2.1.0-r1` and
`luci-app-airoha-flowsense 1.1.3-r3`.

## Reproducible image generation

The custom device profile is named `gemtek_w1700k-ubi2`, while the upstream
U-Boot recipe's automatic default is tied to `gemtek_w1700k-ubi`. The profile
now explicitly selects `u-boot-an7581_gemtek_w1700k`. The workflow builds and
checks the compressed U-Boot image and DTB before the global image build.

The successful run generated both the UBI2 sysupgrade image and
`openwrt-airoha-an7581-gemtek_w1700k-ubi2-chainload-uboot.itb`. The chainload
artifact is not a release asset, to avoid presenting it as a sysupgrade image.

## Checks passed

The seven release assets were downloaded from GitHub and checked independently
with `scripts/audit-w1700k-custom-v2-image.sh`. The audit passed:

- OpenWrt checksums, image size, profile, board, source and Linux `6.18.38`;
- GitHub build-provenance attestations for all seven published assets;
- FIT kernel, DTB, rootfs, UBI2 sysupgrade metadata and 64 KiB ramoops node;
- package manifest and CycloneDX SBOM policy;
- exact builder, container, OpenWrt, Gilly, feed and source-overlay provenance;
- root-cause patch `971` present and fallback patch `965` absent;
- full `wpad-openssl`, DoQ recovery, hostapd-aware Wi-Fi recovery and BusyBox
  `flock` serialization;
- RTL8261CE matcher hash, NPU/MT7996 firmware, radar fix, normal dual-stack fw4
  rule and hardware-offload policy;
- FlowSense opt-in probe, irqbalance, conservative Watchcat, SQM/Cake disabled
  by default, pstore, SFTP and the selected administration tools;
- explicit W1700K U-Boot selection in the effective build configuration.

No backup, `raw/` capture or tar archive was published. The macOS extraction
warning for the rootfs `/dev/console` character device is host-only; all
regular files and policy checks completed, and Linux CI validated the assembled
rootfs before publication.

## Remaining hardware gate

This proves image structure, provenance and contents, not runtime behavior. The
active router still runs the validated Gilly 17.07 image. A separate, explicit
authorization is required before testing v2.1 on hardware.

That test must still cover boot and rollback, RTL8261CE binding on WAN/LAN2,
WAN 10G, multigig LAN2, sustained IPv4/IPv6 PPE traffic, all three APs, DoQ,
Watchcat, pstore and the irqbalance comparison. Until then, v2.1 remains an
untested prerelease rather than the recommended production image.
